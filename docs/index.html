codonyat · RNA-temporal variant tracking

codon-yat — codon-aware amino acid variant typing

codonyat validates SAM/reference/amplicon triples, loads every mapped read into a `SamContainer`, and emits strand-aware ratios, entropy, and TSV/XML diagnostics that automation pipelines consume without extra parsing.

Highlights

  • Produces the same TSV/XML diagnostics as the legacy Perl outputs while exposing Python objects (`SamContainer`, `FullReference`, etc.) for tooling.
  • Validates every input file before parsing so malformed SAM/FASTA/amplicon data fail fast.
  • Records per-position entropy and strand balance for CI-friendly debugging and auditing.

Installation

Install the package locally (or publish via PyPI/Twine) before running the CLI or importing it:

pip install .

The `codonyat` entry point appears on your PATH once installation completes.

CLI usage

Invoke the tool with a SAM file, a FASTA reference, and an amplicon TSV:

codonyat sample.sam reference.fasta amplicons.tsv

Tune the strand-ratio window and entropy threshold as needed:

codonyat sample.sam reference.fasta amplicons.tsv --ratio-upper 3.16 --ratio-lower 0.32 --entropy-threshold 0.5

The CLI emits sample.sam.tsv and sample.sam.xml with columns for FILE, REFERENCE, PROTEIN, VARIANT, POSITION, FREQ, FWCOV, RVCOV, TOTALCOV, and RATIO so downstream consumers get consistent diagnostics.

The lightweight codonyat-runner entry point reuses the parsed arguments, supports --csv-path/--xml-path overrides, and prints the generated artifact locations so it fits into leaner pipelines.

codonyat-runner sample.sam reference.fasta amplicons.tsv --ratio-upper 3.1 --csv-path results.tsv

Package API

Import the core objects when you need tighter integration:

from aa_caller import SamContainer, FullReference, parse_amplicons

Use call_variants to validate inputs, run the translation pipeline, emit the TSV/XML artifacts, and return a VariantCallResult with the paths plus the underlying SamContainer:

from pathlib import Path

from aa_caller import call_variants

result = call_variants(
  sam_path=Path("reads.sam"),
  reference_path=Path("reference.fasta"),
  amplicons_path=Path("amps.tsv"),
  ratio_upper=3.2,
)

print(result.csv_path, result.xml_path)
print(result.container.variants.keys())

If you have an argparse.Namespace or mapping of CLI arguments, call_variants_from_args adapts them so wrappers like codonyat-runner stay synchronized:

from argparse import Namespace

args = Namespace(
  sam_file="reads.sam",
  reference_file="reference.fasta",
  amplicons_file="amps.tsv",
  ratio_upper=3.2,
)

call_variants_from_args(args)

Development

Install the repo with optional dev tooling to match CI:

pip install --upgrade pip
pip install -e .[dev]

The workflow installs pytest and ruff and runs them on every push/PR; see the CI definition for details.

Testing

  • Validation helpers cover codon translation, input validation, and simple parser paths.
  • The pytest suite lives in tests/test_app.py and keeps the legacy behavior locked down.
  • Run python -m pytest locally, preferably after ruff check ..
